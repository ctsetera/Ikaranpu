# セキュリティポリシー / Security Policy

## 日本語

### サポート対象バージョン

セキュリティアップデートは、本アプリの最新リリース版を対象に提供します。

過去のバージョンには、セキュリティ修正が提供されない場合があります。可能な限り最新バージョンへ更新してください。

### 脆弱性の報告

セキュリティ上の脆弱性を発見した場合は、公開 GitHub Issues には報告しないでください。

GitHub の private vulnerability reporting 機能を使用して、非公開で報告してください。

報告時には、可能な範囲で以下の情報を含めてください。

- 問題の概要
- 再現手順
- 影響を受けるアプリのバージョン
- 影響を受ける端末および Android バージョン
- 関連するスクリーンショット、ログ、または検証用の情報
- 想定される影響範囲

### 対応方針

報告を受け取った後、メンテナーが内容を確認し、影響範囲を調査します。

問題が確認された場合は、必要に応じて修正を行い、リリースします。修正が提供されるまで、詳細の公開は調整のうえ行ってください。

### 対象範囲

このポリシーは、本アプリおよびそのソースコードに関するセキュリティ上の問題を対象とします。

サードパーティサービス、外部 API、Firebase、Google Play、Android、その他の依存関係に関する問題については、必要に応じて各提供元にも報告してください。

### 注意事項

ユーザーへの被害、データの侵害、外部サービスへの過剰な負荷、または適用される法律や利用規約への違反につながる行為は避けてください。

## English

### Supported Versions

Security updates are provided for the latest released version of this app.

Older versions may not receive security fixes. Please update to the latest version when possible.

### Reporting a Vulnerability

If you discover a security vulnerability, please do not report it through public GitHub Issues.

Please report it privately using GitHub's private vulnerability reporting feature.

When reporting a vulnerability, please include as much of the following information as possible:

- A description of the issue
- Steps to reproduce the issue
- The affected app version
- The affected device and Android version
- Any relevant screenshots, logs, or proof-of-concept details
- Your assessment of the potential impact

### Handling Process

After receiving a report, the maintainer will review the issue and investigate its impact.

If the issue is confirmed, a fix will be prepared and released as appropriate. Public disclosure should be coordinated until a fix is available.

### Scope

This policy applies to security issues in this app and its source code.

Issues related to third-party services, external APIs, Firebase, Google Play, Android, or other dependencies should also be reported to the relevant provider when appropriate.

### Notes

Please avoid actions that may harm users, compromise data, overload external services, or violate applicable laws or terms of service.
